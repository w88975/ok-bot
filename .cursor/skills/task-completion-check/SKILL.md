## 1. 检查描述文件
例如: 新建了一个组件, 按照我们的规范, 在每个组件的目录下, 都会有一个 `README.md` 和 `UI.md` 文件, 该文件描述了该组件的用途, 使用方法, 示例代码, 注意事项等.
当我们对组件进行修改, 或者功能调整后, 必须要及时更新该文件, 以确保该组件的文档是最新的, 并且是准确的.

如果目录下不存在该文件(README.md/UI.md), 则需要创建该文件, 并按照规范填写内容.
