## 新增需求

### 需求:布局结构 — 侧边栏 + 主面板
Web UI 必须提供三列布局：左侧 agent 列表侧边栏、中间聊天主面板、右侧（可选）群组信息面板。

#### 场景:侧边栏显示所有 agent 和群组
- **当** 页面加载完成，WebSocket 收到 `agent-status`
- **那么** 左侧侧边栏显示当前所有 agent 名称及运行状态徽标，以及已创建的群组列表

#### 场景:点击 agent 切换单聊面板
- **当** 用户点击侧边栏中某个 agent
- **那么** 主面板切换为与该 agent 的单聊界面，显示该 agent 的历史消息

---

### 需求:单聊面板
单聊面板必须支持与单个 agent 的实时对话，消息按时间升序显示，最新消息在底部。

#### 场景:发送消息并显示回复
- **当** 用户在输入框输入内容后按 Enter 或点击发送
- **那么** 用户消息立即显示，输入框清空，agent 回复在收到后追加显示（带角色名和时间戳）

#### 场景:发送中显示 loading 状态
- **当** 消息已发送但 agent 尚未回复
- **那么** 界面显示"正在输入..."或 loading 动画

#### 场景:消息过多时自动滚动到底部
- **当** 新消息到达
- **那么** 聊天面板自动滚动到最新消息

---

### 需求:群组聊天面板
群组面板必须在同一视图中展示多个 agent 的回复，每条消息标注发言者（用户 or agent 名称）。

#### 场景:用户消息在群组中广播
- **当** 用户在群组面板发送消息（不带 @mention）
- **那么** 消息发送给群组内所有 agent，各 agent 回复独立追加到消息流

#### 场景:@mention 只路由到指定 agent
- **当** 用户发送 `@agent-a 你好`
- **那么** 仅 agent-a 收到该消息并回复，其他 agent 不响应

#### 场景:多 agent 并发回复的展示
- **当** 群组内多个 agent 并发返回回复
- **那么** 各回复按到达顺序追加，每条均标注对应 agent 名称和头像色块

---

### 需求:创建 Agent 表单
Web UI 必须提供内联创建 agent 的表单，提交后通过 WebSocket 发送 `create-agent` 消息。

#### 场景:填写表单创建 agent
- **当** 用户填写 id、workspace、model、apiKey（可选 baseURL、bootstrap 内容）并提交
- **那么** 发送 `create-agent` 消息，成功后 agent 出现在侧边栏，表单关闭

#### 场景:id 重复时显示错误
- **当** 创建的 agent id 已存在
- **那么** 表单内显示错误提示，不关闭表单

#### 场景:必填字段未填写时禁止提交
- **当** `id`、`workspace` 或 `provider.model` 为空
- **那么** 提交按钮禁用，字段旁显示红色提示

---

### 需求:创建群组
用户必须能在侧边栏通过"新建群组"按钮选择多个 agent 创建群组会话。

#### 场景:选择 agent 创建群组
- **当** 用户点击"新建群组"，选择 2 个以上 agent，填写群组名后确认
- **那么** 群组出现在侧边栏，点击后进入群组聊天面板

#### 场景:群组至少需要 2 个 agent
- **当** 用户仅选择 1 个 agent 尝试创建群组
- **那么** 确认按钮禁用，显示"至少选择 2 个 agent"提示

---

### 需求:WebSocket 连接状态指示
Web UI 必须在界面顶部或侧边栏显示 WebSocket 连接状态（已连接 / 连接中 / 已断开），断开时自动重连。

#### 场景:连接成功显示绿色指示器
- **当** WebSocket 连接建立
- **那么** 显示绿色"已连接"标记

#### 场景:断开时自动重连
- **当** WebSocket 意外断开
- **那么** 显示黄色"重连中..."，每 3 秒尝试重连，最多 5 次；超出次数显示红色"已断开"

---

### 需求:响应式布局与暗色模式
Web UI 必须支持深色模式（跟随系统），并在中等屏幕（≥768px）下保持三列布局可用。

#### 场景:系统暗色模式下界面切换
- **当** 系统切换到暗色模式
- **那么** Web UI 自动切换配色方案
